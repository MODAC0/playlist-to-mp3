## 적용 내용 (app.py)

### 디자인 시스템 (`build_qss(dark)`)

- **글래스 레이어**: 창 배경을 세로 그라디언트로 깔고, 그 위에 반투명 화이트 카드(`rgba(255,255,255,7%)` 다크 / `58%` 라이트)를 겹쳐 글래스 질감 표현. 입력 필드·버튼은 카드 위의 2차 반투명 레이어. Qt는 실시간 배경 블러가 없어 반투명 + 엣지 보더로 에뮬레이션
- **넓은 패딩**: 루트 여백 28px·간격 16px, 카드 내부 20px, 입력 필드 10x12px, 버튼 9x20px, 창 크기 720x760
- **라운드 코너**: 카드 18px, 필드·버튼 10px, 로그 14px
- **컬러**: 애플 액센트 블루(#007aff/#0a84ff), 주 버튼은 블루 수직 그라디언트, 프로그레스 바는 6px 얇은 바 + 블루 가로 그라디언트
- **라이트/다크 자동 대응**: 시스템 팔레트 밝기로 감지(`_apply_style`), PaletteChange 이벤트 시 재적용. 이벤트 처리 중 setStyleSheet 재진입을 피하려고 `QTimer.singleShot(0, …)` 지연 호출
- **레이아웃 재구성**: 설정 앱처럼 카드 3장(URL+미리보기 / 저장 폴더+옵션 / 진행률) + 우하단 주 동작 버튼(macOS 관례) + 로그 콘솔

## 디버깅 노트 (재발 방지)

- **Qt QSS의 `rgba()` 알파는 CSS와 다르다**: 0~1 소수(`rgba(0,0,0,0.5)`)가 아니라 **0~255 정수 또는 %만 인식**. 소수를 쓰면 스타일이 조용히 무시된다 (다크 모드가 적용 안 되던 원인)
- PaletteChange 핸들러 안에서 직접 `setStyleSheet` 호출은 재진입 위험 → 지연 호출로 처리
