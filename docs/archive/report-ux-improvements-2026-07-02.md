# playlist-to-mp3 UI/UX 개선 보고서

- 날짜: 2026-07-02
- 대상: `playlist-to-mp3` (macOS PySide6 GUI)
- 배경: 크래시 수정(report-gui-crash-fix-2026-07-02.md) 직후, 사용 편의 개선 요청. 6개 항목 채택.

## 적용 내용 (app.py)

1. **클립보드 자동 인식**: 앱 실행 시·창 활성화 시 클립보드에서 YouTube URL을 찾아 자동 입력. 사용자가 직접 입력한 값은 덮어쓰지 않음
2. **시작 전 미리보기**: URL 입력 후 0.8초 디바운스 → 백그라운드에서 flat extraction으로 "✔ 제목 · n곡" 표시. 잘못된 URL은 시작 전에 ✖로 안내
3. **전체 진행 표시**: "전체 n/m곡" 라벨 + 전체 진행 바 추가 (기존 바는 "현재 곡" 진행). 워커의 `item_progress(idx, total)` 시그널 신설
4. **완료 요약 + 폴더 열기**: 완료 팝업에 "성공 x곡 · 건너뜀 y곡 · 실패 z곡" 요약과 "폴더 열기" 버튼. 저장 폴더 행에도 상시 "폴더 열기" 버튼. 전부 건너뜀이면 중복 방지 해제 안내 문구 표시 (아카이브 혼란 해소)
5. **설정 기억**: QSettings("upfall", "playlist-to-mp3")로 저장 폴더·음질·중복 방지 상태를 시작 시·종료 시 저장, 다음 실행에 복원
6. **Enter 시작 + 드래그앤드롭**: URL 칸에서 Enter로 즉시 시작, 브라우저에서 링크를 창에 끌어놓기 지원

## 구현 주의점

- 미리보기 스레드 → UI 전달은 `_PreviewBridge(QObject)` 시그널 + 바운드 메서드 슬롯으로만 연결 (람다 연결 시 워커 스레드 직접 실행 → 크래시 재발 위험, 크래시 수정 보고서 참조). 세대(generation) 카운터로 늦게 도착한 이전 요청 응답은 무시
- 실패 곡 집계는 오류 로그 줄 수가 아니라 "전체 - 성공 - 건너뜀" 산술로 계산 (재시도 후 성공한 곡을 실패로 오집계하는 문제 수정). 전체 곡 수를 모르는 경우만 `✖ ERROR` 라인 카운트 폴백
- yt-dlp의 "Deprecated Feature"(Python 3.9 지원 중단 예고) error 라인은 사용자 노이즈라 로그·집계 모두에서 제외
